## New `text-shadow-*` utilities

We've been threatening to add text shadows for at least the last six years and today they are finally here - It's just a pity no one uses them anymore.

Five text shadows have been added to the default theme, from `text-shadow-2xs` to `text-shadow-lg`. They are particularly useful for making headings stand out against a busy background:

Like with any color, you can control the opacity of a text shadow color with an opacity modifier but we've also added the ability to control the opacity of the default text shadow color.

## Mask elements with the `mask-*` utilities

The new `mask-*` utilities let you create interesting effects by masking elements with images, gradients, or SVG paths:

```html
<div class="mask-gradient-to-r from-black to-transparent ...">
  <!-- Content -->
</div>
```

## Fine-grained text wrapping with `overflow-wrap`

The new `overflow-wrap` utilities let you control how text wraps within an element. This is especially useful for long words or URLs that might otherwise break your layout:

```html
<div class="overflow-wrap-break-word ...">
  <p>Supercalifragilisticexpialidocious</p>
</div>
```

## Target input devices with `pointer-*` and `any-pointer-*`

Building interfaces that work great with both touch and mouse input? The new pointer media query variants make it easy to adapt your UI based on the user's input method:

```html
<button class="pointer-coarse:active:bg-blue-600 pointer-fine:hover:bg-blue-600 ...">Click me or tap me</button>
```

Use `pointer-fine` to target devices with precise pointing (like mice), and `pointer-coarse` for touch input.

## Keep content visible with `safe` alignment

Ever had content scroll off the screen because of an unfortunate alignment? The new safe alignment utilities help prevent that:

```html
<div class="justify-start-safe">
  <!-- Content won't get cut off -->
</div>
```

These utilities work with both flexbox and grid layouts, and are available for all alignment properties.

## Safelist specific utilities with `@source inline(…)`

Need to include utilities that aren't in your source files? Use `@source inline`:

```css
@source inline(.bg-purple-600, .text-3xl);
```

## Ignore specific paths with `@source not`

Control which paths Tailwind scans for classes using `@source not`:

```css
@source not('test/**/*.test.js');
```

## Target `inverted-colors` mode

For users with inverted colors enabled, you can now target that mode specifically:

```html
<div class="inverted-colors:invert-0">
  <!-- This content won't be inverted in inverted colors mode -->
</div>
```

---

## New `noscript` variant

Need to show something different when JavaScript is disabled? Use the new `noscript` variant:

```html
<div class="hidden noscript:block">Please enable JavaScript to use this app.</div>
```
